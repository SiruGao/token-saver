# Token Saver Desktop V1

## Product definition

Token Saver V1 is a local-first desktop control plane for diagnosing AI-agent token waste and matching findings to compatible third-party compression strategies.

The application is the product. The OpenClaw skill is retained as one optional integration.

```text
Doctor → Policy → Strategy Adapter → Proof
```

V1 delivers the Doctor, policy shortlist, strategy registry, and release visibility. Runtime execution and Proof automation come later.

## Included in V1

- Tauri 2 desktop shell for macOS, Windows, and Linux;
- local UI with Dashboard, Doctor, Strategies, Sessions, Integrations, and Settings;
- read-only detection of common local agent installations;
- explicit JSON, JSONL, nested-object, and text transcript import;
- generic transcript normalization;
- token estimates when provider usage is unavailable;
- provider usage ingestion when common usage fields exist;
- Doctor rules for repeated reads, identical results, oversized output, long instructions, prompt-prefix drift, and possible rework;
- Strategy Hub with:
  - metadata for RTK, Headroom, and Claw Compactor;
  - license, mode, risk, capability, and compatibility declarations;
  - Doctor-driven recommendations;
  - upstream GitHub release checks;
  - user-controlled strategy selection;
  - observe-only adapter contract;
- local persistence and JSON report export;
- embedded app icon and cross-platform CI;
- read-only behavior: V1 does not modify agent configuration, prompts, commands, or workspaces.

## Deliberate limitations

V1 does not yet include:

- third-party strategy installation or execution;
- compatibility-certified updates;
- automatic transcript ingestion;
- a transparent LLM proxy;
- automatic prompt rewriting or compression;
- SQLite storage;
- provider account or billing integrations;
- quality replay or verified cost savings;
- code signing and public release artifacts;
- version-specific parsing for every agent.

A release being visible does not mean it is compatible. Future adapters must pass version constraints and health checks before execution.

## Run

```bash
npm install
npm run dev          # web preview
npm run desktop:dev  # native development
npm run desktop:build
```

Bundles are created under `src-tauri/target/release/bundle`.

## Privacy model

- analysis occurs on the local device;
- no telemetry is implemented;
- no account is required;
- transcript content is processed only after explicit selection;
- native detection checks whether known application directories exist but does not read their files;
- strategy release checks request public GitHub metadata only;
- V1 does not automatically traverse agent configuration or session directories.

## V1 acceptance criteria

- TypeScript strict checking passes;
- the Vite production build passes;
- Rust/Tauri compilation passes;
- the app opens in web preview and desktop mode;
- demo data renders Dashboard, Doctor, Strategy Hub, Sessions, Integrations, and Settings;
- imported transcripts generate sessions and Doctor findings;
- Doctor findings map to compatible strategies;
- strategy release metadata can be refreshed without installing software;
- native mode detects supported agent installations without reading or changing their files;
- clear local data removes the local workspace;
- export creates a portable JSON report.
