# Token Saver Desktop V1

## Product definition

Token Saver V1 is a local-first desktop application for observing and diagnosing token waste across AI-agent sessions.

The application is the product. The OpenClaw skill is retained as one optional integration.

## Included in V1

- Tauri 2 desktop shell for macOS, Windows, and Linux;
- polished local UI with Dashboard, Doctor, Sessions, Integrations, and Settings;
- automatic detection of common local agent directories;
- local scanning of recent JSON, JSONL, and text session files;
- drag-and-drop or file-picker transcript import;
- generic transcript normalization;
- token estimates when provider usage is unavailable;
- provider usage ingestion when common usage fields exist;
- Doctor rules for:
  - repeated file reads;
  - identical repeated tool results;
  - oversized tool output;
  - long persistent instruction blocks;
  - unstable prompt prefixes;
  - possible tool-call rework;
- local persistence and JSON report export;
- read-only behavior: V1 does not modify agent configuration or prompts.

## Deliberate limitations

V1 is an analyzer, not yet a universal runtime optimizer.

It does not yet include:

- a transparent LLM proxy;
- automatic prompt rewriting;
- SQLite storage;
- provider authentication or billing APIs;
- verified cost savings;
- automatic configuration changes;
- code-signing and public release artifacts;
- adapter-perfect parsing for every agent transcript version.

These limitations are explicit so the project does not claim savings that have not been measured.

## Run the web preview

```bash
npm install
npm run dev
```

## Run the desktop application

Install the platform prerequisites described by Tauri, then:

```bash
npm install
npm run desktop:dev
```

## Build installers

```bash
npm install
npm run desktop:build
```

Tauri creates the platform-specific bundle under `src-tauri/target/release/bundle`.

## Privacy model

- analysis occurs on the local device;
- no telemetry is implemented;
- no account is required;
- imported content is stored only in local app storage;
- the native scanner reads only JSON, JSONL, and text files under detected agent directories;
- file size, count, and traversal depth are capped.

## V1 acceptance criteria

- the TypeScript project passes `npm run check`;
- the frontend builds with Vite;
- the app opens in web preview and desktop mode;
- demo data renders all main views;
- imported transcripts generate sessions and Doctor findings;
- native mode detects supported agent directories without changing them;
- clear local data removes the local workspace;
- export creates a portable JSON report.
